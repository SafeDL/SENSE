# 实验设计与实现细节

## 3. 实验设计

### 3.1 基准测试环境

在自动驾驶场景搜索任务上进行评估，使用代理模型（Surrogate Model）作为黑盒优化目标：

$$f(\mathbf{x}) = \text{GP}(\mathbf{x}; \theta_{\text{GP}})$$

其中GP为高斯过程回归模型，在真实CARLA仿真数据上训练。

**搜索空间**：3维场景参数空间 $\mathcal{X} = [-1, 1]^3$
- 参数1：目标车辆初始速度
- 参数2：障碍物位置
- 参数3：交通流密度

**危险阈值**：$\tau_d = -0.3$（归一化安全评分）

### 3.2 对比方法

1. **随机搜索 (Random Search)**：均匀随机采样
2. **遗传算法 (GA)**：标准遗传算法，种群100
3. **贝叶斯优化 (BO)**：高斯过程+EI采集函数
4. **标准PSO**：单种群PSO，参数固定
5. **小生境PSO (Niche PSO)**：多子种群，参数固定
6. **PPO-PSO**：使用PPO Agent控制PSO参数（基线）
7. **SAC-PSO**（本方法）：使用SAC Agent控制PSO参数

### 3.3 评估指标

**1. 覆盖率 (Coverage Rate)**
$$\text{Cov} = \frac{|\{g_i : \exists \mathbf{x} \in g_i, f(\mathbf{x}) < \tau_d\}|}{m}$$

其中网格划分为 $m=97$ 个单元（基于训练数据统计）。

**2. 最优适应度 (Best Fitness)**
$$f_{\text{best}} = \min_{i=1}^{N} f(\mathbf{x}_i)$$

**3. 危险场景发现数 (Hazard Count)**
$$N_{\text{hazard}} = |\{\mathbf{x} : f(\mathbf{x}) < \tau_d\}|$$

**4. 函数评估次数 (Function Evaluations)**
$$\text{FE} = \text{总评估次数}$$

**5. 计算效率 (Efficiency)**
$$\text{Eff} = \frac{\text{Cov}}{\text{FE}}$$

### 3.4 实验配置

**SAC-PSO参数**：
- 粒子数：500
- 子种群数：10
- 最大迭代：200
- 动作间隔 $K$：10
- 动作平滑因子 $\alpha$：0.6

**SAC Agent参数**：
- 状态维度：28
- 动作维度：4（连续：$w, c_1, c_2, v_{\text{scale}}$）
- 隐层维度：[128, 128, 64]
- 学习率：$3 \times 10^{-4}$
- 折扣因子 $\gamma$：0.7
- 软更新系数 $\tau$：0.005
- 初始温度 $\alpha$：0.05
- 批大小：256
- 回放缓冲区大小：100,000

**训练设置**：
- 独立运行次数：5次
- 随机种子：[42, 123, 456, 789, 1024]
- 每次运行的Episode数：500

## 4. 关键实现细节

### 4.1 动作空间映射

**连续动作到PSO参数的映射**：

SAC输出的原始动作 $\mathbf{a}_{\text{raw}} \in [-1, 1]^4$ 通过线性缩放映射到PSO参数范围：

$$\mathbf{a}_{\text{scaled}} = \mathbf{a}_{\text{raw}} \cdot \mathbf{s} + \mathbf{b}$$

其中缩放向量和偏置向量为：

| 参数 | 范围 | 缩放 $s$ | 偏置 $b$ |
|------|------|---------|---------|
| $w$ (惯性权重) | [0.4, 0.9] | 0.25 | 0.65 |
| $c_1$ (认知系数) | [0.5, 2.5] | 1.0 | 1.5 |
| $c_2$ (社会系数) | [0.5, 2.5] | 1.0 | 1.5 |
| $v_{\text{scale}}$ (速度缩放) | [0.5, 2.0] | 0.75 | 1.25 |

### 4.2 平滑动作执行

为避免参数剧烈波动导致搜索不稳定，采用指数平滑更新：

$$\theta_g^{(t+1)} = (1-\alpha) \theta_g^{(t)} + \alpha \theta_{\text{new}}$$

其中 $\alpha = 0.6$ 为平滑因子。这确保参数变化平缓，同时保留Agent的控制信号。

### 4.3 宏命令缓冲区管理

为处理并行子种群的异步转移，使用独立的轨迹缓冲区：

```
group_trajectory_buffers[g] = [(s, a, r, s', log_prob, value), ...]
```

**决策步** ($t \equiv 0 \pmod{K}$)：
- 将上一个宏命令的累积转移存入缓冲区
- 采样新动作，重置累积奖励

**执行步** ($t \not\equiv 0 \pmod{K}$)：
- 累积单步奖励
- 维持当前动作

**Episode结束**：
- 强制刷新所有缓冲区到Agent的回放缓冲区
- 标记最后一步为终止状态

### 4.4 重启机制

当子种群停滞超过阈值时触发重启：

$$\text{Restart if } \text{Stagnation}_g \geq \text{patience} = 5$$

重启过程：
1. 刷新该组的轨迹缓冲区（标记为Done）
2. 随机重新初始化粒子位置和速度
3. 重置PSO参数为默认值
4. 重置停滞计数器
5. 增加重启计数器

### 4.5 奖励计算的数值稳定性

为防止奖励爆炸，使用Tanh压缩：

$$r_{\text{final}} = \tanh(r_{\text{raw}})$$

这将奖励限制在 $[-1, 1]$ 范围内，同时保留相对大小关系。

### 4.6 状态特征的无量纲化

所有状态特征都进行无量纲化处理，确保网络输入的数值稳定性：

**变异系数**：
$$CV = \tanh\left(\frac{\sigma}{|\mu| + \epsilon}\right) \in [-1, 1]$$

**改进率**：
$$\text{Imp}_{\text{norm}} = \tanh(10 \cdot \text{Imp}_{\text{raw}}) \in [-1, 1]$$

**距离比**：
$$d_{\text{norm}} = \frac{d}{\sqrt{d}} \in [0, 1]$$

## 5. 理论分析

### 5.1 收敛性保证

**定理 1（SAC的渐近收敛性）**：
在标准假设下（有界奖励、紧凑状态空间、连续策略），SAC算法的策略序列 $\{\pi_k\}$ 以概率1收敛到最优策略 $\pi^*$。

*证明思路*：SAC通过最大化下界 $J(\pi) = \mathbb{E}[r + \alpha H(\pi)]$ 进行优化，该下界是原始目标的凸组合。通过软更新机制和双Q网络的高估缓解，保证了单调改进性质。

### 5.2 样本效率分析

**命题 1（宏命令的样本效率）**：
相比单步决策，宏命令执行策略将样本复杂度降低 $O(K)$ 倍，其中 $K$ 为动作间隔。

*证明*：每 $K$ 步产生一个转移样本，而单步决策产生 $K$ 个样本。在相同的函数评估次数下，宏命令方法的转移数为 $\frac{1}{K}$，但每个转移的累积奖励包含 $K$ 步的信息，信息密度相同。

### 5.3 多样性维护

**引理 1（小生境的多样性保证）**：
在增强NDM算子的作用下，子种群间的最小距离满足：

$$\mathbb{E}[d_{\min}^{(t+1)}] \geq (1 - \delta) d_{\min}^{(t)}$$

其中 $\delta$ 为衰减因子，取决于NDM触发频率。

*直观解释*：NDM通过差分变异引入新的搜索方向，防止子种群过度聚集。

## 6. 实现优化

### 6.1 GPU加速

所有张量操作使用PyTorch在GPU上执行：
- 粒子位置/速度：GPU张量
- 适应度评估：批量CPU评估，结果转移到GPU
- 网络前向/反向传播：GPU执行

### 6.2 批量操作

PSO更新使用向量化操作：

```python
# 批量速度更新 (N个粒子)
v = w * v + c1 * r1 * (pbest - x) + c2 * r2 * (gbest - x)

# 批量位置更新
x = (x + v).clamp(bounds[0], bounds[1])
```

相比逐粒子更新，速度提升 10-50 倍。

### 6.3 内存管理

- 回放缓冲区使用循环队列，固定内存占用
- 轨迹缓冲区在Episode结束后清空
- 梯度计算后立即释放中间张量
